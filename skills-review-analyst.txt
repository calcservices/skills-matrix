========================
Levels: Foundation → Proficient → Advanced → Expert
Format: each level shows what it looks like, followed by how to progress to the next.

CATEGORY: Specification & Analysis
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─────────────────────────────────────────────────────────────────────────────
│ Writing Calc Specifications
│ Producing clear, complete, and unambiguous calculation specifications that
│ developers can code from — covering all scenarios, edge cases, and business
│ rules.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can write simple spec sections with guidance. Understands the basic spec
    template and structure.
  → To reach Proficient: Study completed specs alongside the finished code. See how developers interpreted your words.

  Proficient
    Writes standard calc specs independently. Covers happy-path scenarios and
    common edge cases. Specs are clear and codeable.
  → To reach Advanced: Track the questions developers ask during build. Use these to improve your next spec.

  Advanced
    Writes complex multi-scenario specs. Anticipates developer questions. Handles
    interactions between calc types and scheme-specific nuances.
  → To reach Expert: Spec a complex calc (e.g. GMP equalisation) end-to-end. Get peer review from a senior analyst.

  Expert
    Defines spec standards and templates. Reviews other analysts' specs. Drives
    continuous improvement in spec quality across the team.


┌─────────────────────────────────────────────────────────────────────────────
│ Requirements Gathering
│ Eliciting, documenting, and validating requirements from scheme rules,
│ legislation, client instructions, and stakeholder conversations.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can extract basic requirements from scheme rules with guidance. Takes notes
    in meetings.
  → To reach Proficient: Sit in on requirements sessions. Compare the rules to the final spec to see what was extracted.

  Proficient
    Gathers requirements independently from rules and client instructions. Asks
    clarifying questions. Documents requirements clearly.
  → To reach Advanced: Lead a requirements gathering session for a straightforward calc. Prepare questions in advance.

  Advanced
    Handles ambiguous or conflicting requirements. Facilitates discovery
    workshops. Identifies unstated assumptions.
  → To reach Expert: Handle a scheme with ambiguous rules. Document your assumptions and get sign-off.

  Expert
    Defines the requirements process. Coaches others on elicitation techniques.
    Manages complex multi-stakeholder requirements.


┌─────────────────────────────────────────────────────────────────────────────
│ Gap Analysis & Impact Assessment
│ Identifying what needs to change when scheme rules are amended, legislation
│ changes, or new requirements emerge — and assessing the impact on existing
│ calcs.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can compare two versions of a document to spot differences with guidance.
  → To reach Proficient: Compare a rule amendment to the existing spec. List every difference you find.

  Proficient
    Performs gap analysis independently. Identifies affected calcs and documents
    changes needed.
  → To reach Advanced: Perform a full gap analysis for a rule change. Walk through the impact with a developer.

  Advanced
    Assesses cross-scheme impact. Identifies knock-on effects and dependencies.
    Produces clear impact reports.
  → To reach Expert: Handle a legislative change affecting multiple schemes. Create a reusable impact template.

  Expert
    Defines the impact assessment framework. Advises on complex regulatory
    changes affecting multiple schemes.


┌─────────────────────────────────────────────────────────────────────────────
│ Data Analysis & Querying
│ Analysing scheme data to support specifications, validate calcs, investigate
│ issues, and produce reports — using SQL, Excel, or other tools.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can work with data in Excel (filters, sorting, basic formulas). Understands
    table structures with guidance.
  → To reach Proficient: Learn basic SQL SELECT statements. Practice with real scheme data in a safe environment.

  Proficient
    Writes basic SQL queries. Creates pivot tables and lookups in Excel. Uses
    data to validate calc outputs.
  → To reach Advanced: Write queries to investigate a support ticket. Build a spreadsheet model to cross-check results.

  Advanced
    Writes complex queries (joins, CTEs, aggregations). Analyses large datasets.
    Produces data-driven insights for spec decisions.
  → To reach Expert: Analyse a full scheme's data for a migration or reconciliation exercise.

  Expert
    Go-to person for data questions. Designs data validation strategies. Builds
    reusable query libraries.

┌─────────────────────────────────────────────────────────────────────────────
│ Scheme Rules & Legislation
│ Deep understanding of pension scheme rules, trust deeds, overriding
│ legislation, and how they interact to define calculation requirements.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Aware that scheme rules exist. Can read a rule with guidance to understand a
    calc requirement.
  → To reach Proficient: Read the rules for your current scheme alongside the spec. Ask your lead to explain key sections.

  Proficient
    Reads and interprets rules independently for standard calcs. Understands key
    legislation (GMP, LTA, AA, etc.).
  → To reach Advanced: Study the Pensions Act basics and key statutory instruments. Read rules for multiple schemes to see patterns.

  Advanced
    Interprets complex or ambiguous rules. Spots conflicts between rules and
    legislation. Advises on edge cases and precedent.
  → To reach Expert: Attend industry webinars. Study complex scenarios like GMP equalisation and McCloud.

  Expert
    Deep legislative knowledge. Consulted on rule interpretation across schemes.
    Keeps team updated on regulatory changes.


┌─────────────────────────────────────────────────────────────────────────────
│ Calculation Types & Methodologies
│ Understanding the full range of pension calculations — quotes, retirements,
│ deaths, transfers, refunds, augmentations — and the methodologies behind them.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Understands 2-3 basic calc types (e.g. deferred quote, retirement). Can
    follow the methodology with guidance.
  → To reach Proficient: Work through a simple calc type end-to-end — from rule to spec to worked example.

  Proficient
    Comfortable with most standard calc types. Understands how they relate to
    each other and to the benefit structure.
  → To reach Advanced: Expand to less familiar calc types. Understand when each applies and how they differ.

  Advanced
    Handles all calc types including complex ones (transfers, bulk calcs,
    augmentations). Understands actuarial factors and their application.
  → To reach Expert: Spec a complex or unusual calc type. Study actuarial factor tables and their derivation.

  Expert
    Expert across all calc types and methodologies. Defines how new or unusual
    calcs should be approached. Advises actuaries on implementation.


┌─────────────────────────────────────────────────────────────────────────────
│ GMP / Equalisation / Revaluation
│ Specialist knowledge of Guaranteed Minimum Pension, GMP equalisation
│ methodologies, and revaluation — among the most complex areas in UK pensions.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Aware that GMP exists and is complex. Can follow GMP-related spec content
    with heavy guidance.
  → To reach Proficient: Read an introductory guide to GMP. Ask a GMP specialist to explain the basics.

  Proficient
    Understands basic GMP concepts (pre/post-97, fixed/s148 revaluation). Can
    spec guided GMP calcs.
  → To reach Advanced: Study the different revaluation methods. Work on a GMP scheme spec with mentoring.

  Advanced
    Specs GMP equalisation methods (C2, D2, etc.) independently. Handles complex
    revaluation scenarios.
  → To reach Expert: Spec a full GMP equalisation calc. Study the DWP guidance and worked examples in detail.

  Expert
    Expert in all GMP methodologies. Consulted across teams. Keeps up with
    evolving DWP guidance.


┌─────────────────────────────────────────────────────────────────────────────
│ Tax, Limits & Allowances
│ Understanding HMRC tax rules, lifetime and annual allowances, pension
│ commencement lump sums, and how they constrain or modify calculations.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Aware that tax rules apply to pensions. Can follow tax-related spec sections
    with guidance.
  → To reach Proficient: Read the HMRC pensions tax manual basics. Follow a tax calc through a worked example.

  Proficient
    Understands standard tax rules (PCLS, LTA, AA). Specs tax calculations for
    common scenarios.
  → To reach Advanced: Spec the tax elements of a retirement calc. Understand how LTA and AA interact.

  Advanced
    Handles complex tax scenarios (multiple crystallisation events,
    scheme-specific protections, carry forward). Spots tax implications in other
    calcs.
  → To reach Expert: Handle a complex tax scenario (e.g. multiple protections, serious ill-health). Build reference examples.

  Expert
    Expert in pension taxation. Advises on novel tax scenarios. Keeps up with
    HMRC changes and budget impacts.

CATEGORY: Testing & Validation
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─────────────────────────────────────────────────────────────────────────────
│ Worked Examples & Spreadsheets
│ Creating manually calculated worked examples and spreadsheet models to
│ validate that calculation logic is correct before and during development.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can follow a worked example. Checks code output against expected results with
    guidance.
  → To reach Proficient: Manually work through a simple calc with a calculator to understand each step.

  Proficient
    Creates worked examples from specs. Builds validation spreadsheets.
    Reconciles differences with code output.
  → To reach Advanced: Build your own spreadsheet for a calc you've specced. Compare every intermediate step with the developer.

  Advanced
    Builds complex spreadsheet models for multi-step calcs. Identifies errors in
    both code and examples. Creates edge-case scenarios.
  → To reach Expert: Create a library of validation spreadsheets for common calc types.

  Expert
    Designs the worked example process. Creates reusable validation spreadsheets.
    Trains others on spreadsheet modelling.


┌─────────────────────────────────────────────────────────────────────────────
│ UAT Coordination & Support
│ Planning, coordinating, and supporting User Acceptance Testing — working with
│ admin teams to validate that calculations are correct and fit for purpose.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Understands what UAT is. Can respond to simple UAT queries with guidance.
  → To reach Proficient: Sit in on a UAT session. See how admin users interact with the calcs.

  Proficient
    Supports UAT independently. Explains calc results clearly to admin users.
    Helps investigate discrepancies.
  → To reach Advanced: Own UAT support for a scheme. Create a test pack before UAT starts.

  Advanced
    Designs UAT test plans. Anticipates admin questions. Coordinates UAT across
    multiple calcs or schemes. Streamlines the process.
  → To reach Expert: Design a comprehensive UAT plan covering all scenarios. Gather admin feedback and act on it.

  Expert
    Defines UAT standards. Builds tools or templates to support admin testing.
    Transforms UAT efficiency.


┌─────────────────────────────────────────────────────────────────────────────
│ Test Strategy & Scenarios
│ Designing comprehensive test scenarios that cover all paths through a
│ calculation — normal, boundary, edge cases, and error conditions.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can identify basic test scenarios (happy path, simple variants) with
    guidance.
  → To reach Proficient: For your next spec, list every scenario you can think of before looking at existing tests.

  Proficient
    Designs test scenarios independently covering common edge cases. Thinks about
    boundary values and date scenarios.
  → To reach Advanced: Study boundary value analysis and equivalence partitioning concepts. Apply them to a calc.

  Advanced
    Creates comprehensive test matrices. Identifies complex interactions between
    scenarios. Tests for regulatory boundary conditions.
  → To reach Expert: Create a full test matrix for a complex calc. Review it with a developer for coverage gaps.

  Expert
    Defines test strategy for the team. Ensures systematic coverage. Builds
    scenario libraries for common calc types.


┌─────────────────────────────────────────────────────────────────────────────
│ Proformas & Output Validation
│ Specifying and validating the formatted output documents that present
│ calculation results to scheme members and administrators.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Understands what a proforma is. Can check output against a template with
    guidance.
  → To reach Proficient: Compare a proforma spec to the generated output. Understand every field mapping.

  Proficient
    Specs proforma layouts from templates. Validates field mappings and
    conditional content. Spots formatting issues.
  → To reach Advanced: Spec a proforma end-to-end. Validate the output with sample data.

  Advanced
    Designs complex proformas with conditional sections and multi-scenario
    outputs. Ensures regulatory compliance of output wording.
  → To reach Expert: Handle a complex proforma with conditional sections across multiple scenarios.

  Expert
    Defines proforma standards. Advises on output best practice. Reviews proforma
    specs across schemes.

    Completes assigned tasks within a project. Follows the delivery process with

  → To reach Proficient: Track your estimates vs actuals. Discuss variances with your lead.

  Proficient
    Estimates own work reasonably. Manages own deliverables. Communicates

  → To reach Advanced: Take on more estimation responsibility. Learn to break work into smaller deliverables.

  Advanced
    Plans and delivers complex multi-calc projects. Identifies risks and

  → To reach Expert: Lead the analyst workstream on a project end-to-end. Practice stakeholder communication.

  Expert
    Runs major delivery programmes. Defines estimation frameworks. Mentors others

┌─────────────────────────────────────────────────────────────────────────────
│ Stakeholder Management
│ Managing relationships with clients, actuaries, admin teams, developers, and
│ project managers — balancing competing priorities and expectations.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Communicates clearly within the immediate team. Asks for help when needed.
  → To reach Proficient: Practice summarising requirements in plain English for different audiences.

  Proficient
    Communicates confidently with developers and admin. Writes clear emails and

  → To reach Advanced: Own communication with developers for your scheme. Present progress in team meetings.

  Advanced
    Manages client-facing relationships. Handles difficult conversations.

  → To reach Expert: Lead a client requirements workshop. Get feedback on your communication style.

  Expert
    Influences at a senior level. Represents the team in client meetings and

┌─────────────────────────────────────────────────────────────────────────────
│ Developer Handover & Collaboration
│ Handing over specifications to developers effectively — ensuring they
│ understand the requirements, answering questions, and collaborating
│ throughout the build.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Hands over specs to developers. Answers basic questions about the spec
    content.
  → To reach Proficient: Sit in on a developer's spec review. Note what they focus on and what they question.

  Proficient
    Conducts spec walkthrough sessions. Anticipates developer questions.
    Available and responsive during the build phase.
  → To reach Advanced: Run a formal walkthrough for your next spec. Gather feedback from the developer afterwards.

  Advanced
    Collaborates closely with developers throughout. Helps resolve ambiguities in
    real time. Reviews code output against spec intent.
  → To reach Expert: Pair with a developer during the build. Understand how your spec translates to code.

  Expert
    Defines the handover process. Builds strong analyst-developer partnerships.
    Drives improvements in how specs are consumed.


┌─────────────────────────────────────────────────────────────────────────────

└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Documents own work when asked. Follows existing documentation patterns.
  → To reach Proficient: Document something you learned about a scheme this week. Share it with one person.

  Proficient
    Writes clear scheme notes and process docs proactively. Shares learnings in

  → To reach Advanced: Write a scheme guide for a scheme you know well. Get feedback from someone unfamiliar with it.

  Advanced
    Creates comprehensive scheme guides. Runs knowledge-sharing sessions.

  → To reach Expert: Run a lunch-and-learn session on a domain topic. Create a decision log for a complex spec.

  Expert
    Defines documentation standards. Builds a team knowledge base. Champions

┌─────────────────────────────────────────────────────────────────────────────
│ Support & Live Issue Investigation
│ Investigating support tickets for live pension schemes — analysing calculation

└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Investigates simple tickets with guidance. Can check calc results against
    rules.
  → To reach Proficient: Shadow someone investigating a support ticket. Learn the common issue patterns.

  Proficient
    Resolves standard tickets independently. Traces discrepancies through the

  → To reach Advanced: Take on more tickets independently. Build a personal knowledge base of common issues.

  Advanced
    Handles complex cross-calc issues. Identifies root causes vs symptoms.
    Proposes spec corrections or rule clarifications.
  → To reach Expert: Analyse ticket trends for a scheme. Propose spec improvements to eliminate recurring issues.

  Expert
    Defines support investigation processes. Reduces ticket volume through spec
    improvements. Handles critical incidents.
