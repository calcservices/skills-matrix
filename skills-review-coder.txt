CATEGORY: Technical (C# & Engineering)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

┌─────────────────────────────────────────────────────────────────────────────
│ C# & .NET
│ Proficiency in C# language features, .NET framework/class libraries, and
│ writing clean, maintainable code for pension calculation engines.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Writes basic C# with guidance. Understands variables, loops, conditionals,
    and simple classes.
  → To reach Proficient: Pair with a senior developer. Work through MS Learn C# path.

  Proficient
    Writes well-structured code independently. Comfortable with LINQ, generics,
    async/await, and common patterns.
  → To reach Advanced: Read 'C# in Depth'. Tackle increasingly complex calc modules solo.

  Advanced
    Designs complex class hierarchies. Leverages advanced features (expression
    trees, reflection, spans). Optimises for performance.
  → To reach Expert: Contribute to shared libraries. Mentor juniors on patterns. Profile and optimise hot paths.

  Expert
    Defines coding standards for the team. Deep knowledge of CLR internals.
    Evaluates and adopts new .NET features.


┌─────────────────────────────────────────────────────────────────────────────
│ Reusable Code & Patterns
│ Ability to write DRY, reusable code using design patterns appropriate for
│ pension calculation logic — strategy, template method, factory, etc.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Understands the concept of reuse. Can follow existing patterns in the
    codebase.
  → To reach Proficient: Study existing shared code. Ask why things are structured the way they are.

  Proficient
    Extracts common logic into shared methods/classes. Applies basic patterns
    (factory, strategy).
  → To reach Advanced: Look for duplication in your scheme work and extract it. Read 'Head First Design Patterns'.

  Advanced
    Designs reusable components used across multiple schemes. Balances
    abstraction with clarity.
  → To reach Expert: Propose shared components. Get feedback via code review. Refactor legacy code.

  Expert
    Owns and evolves the shared calc library. Creates patterns that the team
    adopts as standards.


┌─────────────────────────────────────────────────────────────────────────────
│ Unit Testing & TDD
│ Writing automated tests, practising test-driven development, and maintaining
│ reliable test suites for calculation code.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Writes basic unit tests with guidance. Understands Assert methods and test
    structure.
  → To reach Proficient: Write tests for existing untested code to build confidence. Pair with someone who does TDD.

  Proficient
    Writes tests independently. Follows Arrange-Act-Assert. Good coverage of
    happy and edge cases.
  → To reach Advanced: Try writing tests first for your next feature. Use test cases from worked examples.

  Advanced
    Practises TDD. Designs testable code. Uses mocking/faking. Writes
    parameterised tests for calc scenarios.
  → To reach Expert: Introduce parameterised tests for scheme-specific calc scenarios. Mentor others on TDD.

  Expert
    Defines team testing strategy. Champions TDD adoption. Builds test
    infrastructure and helpers.


┌─────────────────────────────────────────────────────────────────────────────
│ SQL & Data Access
│ Writing and optimising SQL queries, stored procedures, and understanding how
│ calculation code interacts with the database layer.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Writes basic SELECT/INSERT/UPDATE. Can read simple stored procedures.
  → To reach Proficient: Practice with real scheme data. Use SSMS to explore the database structure.

  Proficient
    Writes joins, subqueries, CTEs. Can modify stored procedures. Understands
    indexes.
  → To reach Advanced: Study execution plans. Take on stored proc work for your schemes.

  Advanced
    Optimises query performance. Designs data access patterns. Writes complex
    stored procs.
  → To reach Expert: Profile slow queries in production. Learn about indexing strategies.

  Expert
    Go-to person for database issues. Designs schema changes.
    Performance-tunes critical queries.


┌─────────────────────────────────────────────────────────────────────────────
│ Source Control & CI/CD
│ Using Git effectively for branching, merging, and code collaboration.
│ Understanding the CI/CD pipeline for building and deploying calc code.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Commits, pushes, pulls. Creates branches. Resolves simple merge conflicts.
  → To reach Proficient: Learn Git basics beyond the IDE. Practice resolving merge conflicts.

  Proficient
    Uses feature branches consistently. Writes good commit messages. Understands
    the build pipeline.
  → To reach Advanced: Read your team's CI/CD config files. Understand what each step does.

  Advanced
    Manages complex merges. Configures build steps. Troubleshoots pipeline
    failures.
  → To reach Expert: Volunteer to fix build failures. Set up a new project in the pipeline.

  Expert
    Designs branching strategy. Maintains and improves CI/CD pipeline. Trains
    team on Git workflows.


┌─────────────────────────────────────────────────────────────────────────────
│ Debugging & Troubleshooting
│ Systematically diagnosing and fixing issues in calculation code, stored
│ procedures, and the wider platform.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Uses breakpoints and step-through debugging. Can identify simple bugs with
    guidance.
  → To reach Proficient: Practice debugging unfamiliar code. Learn to read stack traces thoroughly.

  Proficient
    Diagnoses issues independently. Uses watch windows, conditional breakpoints,
    logs effectively.
  → To reach Advanced: Take on support tickets to build diagnostic skills. Learn SQL Profiler.

  Advanced
    Debugs complex cross-system issues. Reads stack traces quickly. Diagnoses
    production issues from logs.
  → To reach Expert: Volunteer for cross-scheme production issues. Build a personal debugging checklist.

  Expert
    Solves the problems nobody else can. Builds diagnostic tooling. Identifies
    systemic issues.

┌─────────────────────────────────────────────────────────────────────────────
│ Scheme Rules & Legislation
│ Understanding pension scheme rules, overriding legislation, and how they
│ translate into calculation requirements.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Aware that scheme rules exist. Can read a rule with guidance to understand a
    calc requirement.
  → To reach Proficient: Read the rules for your current scheme alongside the spec. Ask your BA to explain sections.

  Proficient
    Reads and interprets rules independently for standard calcs. Understands key
    legislation (GMP, LTA, etc.).
  → To reach Advanced: Study the Pensions Act basics. Read rules for multiple schemes to see patterns.

  Advanced
    Interprets complex/ambiguous rules. Spots conflicts between rules and
    legislation. Advises on edge cases.
  → To reach Expert: Attend industry webinars. Study complex scenarios like GMP equalisation.

  Expert
    Deep legislative knowledge. Consulted on rule interpretation. Keeps team
    updated on regulatory changes.


┌─────────────────────────────────────────────────────────────────────────────
│ Calculation Specifications
│ Reading and interpreting calculation specifications — understanding the
│ business logic they describe and translating it into code.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can follow a simple spec with guidance. Understands basic spec structure.
  → To reach Proficient: Walk through a completed spec + code with someone who built it.

  Proficient
    Reads and codes from specs independently. Asks good clarifying questions.
  → To reach Advanced: Take on a full calc from spec to delivery. Note every question you have to ask.

  Advanced
    Handles complex specs with multiple scenarios. Spots ambiguities and gaps
    before coding.
  → To reach Expert: Review specs before coding starts. Build a checklist of common spec issues.

  Expert
    Reviews specs for completeness. Provides feedback to spec writers. Defines
    spec standards.


┌─────────────────────────────────────────────────────────────────────────────
│ Worked Examples & Spreadsheets
│ Working with manually calculated examples and Excel spreadsheets to verify
│ calculation logic and validate outputs.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Can follow a worked example. Matches code output to expected results with
    guidance.
  → To reach Proficient: Manually work through a simple calc with a calculator to understand each step.

  Proficient
    Creates test cases from worked examples. Reconciles differences between code
    and spreadsheet results.
  → To reach Advanced: Build your own spreadsheet for a calc you're working on. Compare every intermediate step.

  Advanced
    Builds own spreadsheet models to validate complex calcs. Identifies errors in
    provided examples.
  → To reach Expert: Create a library of validation spreadsheets for common calc types.

  Expert
    Designs the worked example process. Creates reusable validation spreadsheets.
    Trains others.


┌─────────────────────────────────────────────────────────────────────────────
│ Proformas & Output Formats
│ Understanding and implementing the formatted output documents that present
│ calculation results to scheme members and administrators.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Understands what a proforma is. Can make minor formatting changes with
    guidance.
  → To reach Proficient: Compare a proforma spec to the generated output. Understand every field mapping.

  Proficient
    Implements proformas from templates. Maps calculation outputs to display
    fields correctly.
  → To reach Advanced: Take ownership of proforma delivery for a scheme. Learn the templating system.

  Advanced
    Handles complex conditional formatting and multi-scenario proformas. Debugs
    layout issues.
  → To reach Expert: Handle a complex proforma with conditional sections. Build a testing approach for layouts.

  Expert
    Designs proforma templates. Advises on output standards. Optimises the
    proforma generation engine.


┌─────────────────────────────────────────────────────────────────────────────
│ GMP / Equalisation / Revaluation
│ Specialist knowledge of Guaranteed Minimum Pension calculations, GMP
│ equalisation methods, and revaluation — among the most complex areas in UK
│ pensions.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Aware that GMP exists and is complex. Can follow GMP-related code with heavy
    guidance.
  → To reach Proficient: Read an introductory guide to GMP. Ask a GMP specialist to explain the basics.

  Proficient
    Understands basic GMP concepts (pre/post-97, fixed/s148 revaluation). Can
    implement guided GMP calcs.
  → To reach Advanced: Study the different revaluation methods. Work on a GMP scheme with mentoring.

  Advanced
    Implements GMP equalisation methods (C2, D2, etc.). Handles complex
    revaluation scenarios.
  → To reach Expert: Implement a full GMP equalisation calc. Study the DWP guidance in detail.

  Expert
    Expert in all GMP methodologies. Consulted across teams. Keeps up with
    evolving guidance.

┌─────────────────────────────────────────────────────────────────────────────
│ Project Delivery
│ Estimating, planning, and delivering calculation work within project
│ timelines — from initial analysis through to go-live.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Completes assigned tasks within a project. Follows the delivery process with
    guidance.
  → To reach Proficient: Track your estimates vs actuals. Discuss variances with your lead.

  Proficient
    Estimates own work reasonably. Manages own deliverables. Communicates
    progress proactively.
  → To reach Advanced: Take on more estimation responsibility. Learn to break work into smaller deliverables.

  Advanced
    Plans and delivers complex multi-calc projects. Identifies risks early.
    Adjusts plans pragmatically.
  → To reach Expert: Lead a small project end-to-end. Practice stakeholder communication.

  Expert
    Runs major delivery programmes. Defines estimation frameworks. Mentors others
    on delivery skills.


┌─────────────────────────────────────────────────────────────────────────────
│ Support & Live Issues
│ Investigating and resolving support tickets for live pension schemes —
│ diagnosing calculation errors, data issues, and urgent fixes.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Investigates simple tickets with guidance. Can read logs and trace basic
    issues.
  → To reach Proficient: Shadow someone resolving a support ticket. Learn the common issue patterns.

  Proficient
    Resolves standard tickets independently. Follows escalation procedures.
    Documents resolutions.
  → To reach Advanced: Take on more tickets independently. Build a personal knowledge base of resolutions.

  Advanced
    Handles complex cross-scheme issues. Identifies root causes vs symptoms.
    Proposes systemic fixes.
  → To reach Expert: Analyse ticket trends. Propose code fixes to eliminate recurring issues.

  Expert
    Defines support processes. Reduces ticket volume through preventive measures.
    Handles critical incidents.


┌─────────────────────────────────────────────────────────────────────────────
│ UAT & Admin Collaboration
│ Supporting the admin team through User Acceptance Testing — explaining calcs,
│ helping reproduce issues, and ensuring smooth handover.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Understands what UAT is. Can respond to simple UAT queries with guidance.
  → To reach Proficient: Sit in on a UAT session. See how admin users interact with the calcs.

  Proficient
    Supports UAT independently. Explains calc results clearly to non-technical
    users. Fixes UAT defects promptly.
  → To reach Advanced: Own UAT support for a scheme. Practice explaining technical concepts simply.

  Advanced
    Designs UAT test plans with admin. Anticipates admin questions. Streamlines
    the UAT process.
  → To reach Expert: Create UAT guides tailored to each scheme. Gather admin feedback and act on it.

  Expert
    Defines UAT standards. Builds tools to support admin testing. Transforms UAT
    efficiency.


┌─────────────────────────────────────────────────────────────────────────────
│ Code Review
│ Giving and receiving constructive code reviews — improving code quality,
│ sharing knowledge, and maintaining standards.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Responds to review feedback positively. Makes requested changes.
  → To reach Proficient: Read review comments on others' PRs to learn what reviewers look for.

  Proficient
    Gives useful review comments on logic, naming, and structure. Understands
    team coding standards.
  → To reach Advanced: Start reviewing PRs proactively. Focus on readability and correctness.

  Advanced
    Reviews complex code effectively. Catches subtle bugs and design issues.
    Mentors through reviews.
  → To reach Expert: Review code in unfamiliar areas to broaden knowledge. Give actionable suggestions, not just criticism.

  Expert
    Defines review standards. Ensures reviews drive team-wide improvement.
    Balances thoroughness with velocity.


┌─────────────────────────────────────────────────────────────────────────────
│ Documentation & Knowledge Sharing
│ Writing clear documentation, sharing knowledge with the team, and ensuring

└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Documents own work when asked. Follows existing documentation patterns.
  → To reach Proficient: Document something you learned this week. Share it with one person.

  Proficient
    Writes clear READMEs and inline comments proactively. Shares learnings in

  → To reach Advanced: Write a how-to guide for a process you know well. Get feedback from someone new.

  Advanced
    Creates comprehensive technical guides. Runs knowledge-sharing sessions.
    Documents architectural decisions.
  → To reach Expert: Run a lunch-and-learn session. Create an ADR for a recent decision.

  Expert
    Defines documentation standards. Builds a team knowledge base. Champions
    documentation culture.

    Communicates clearly within the immediate team. Asks for help when needed.

┌─────────────────────────────────────────────────────────────────────────────
│ Mentoring & Coaching
│ Helping team members develop their skills through mentoring, pairing,
│ coaching, and creating a supportive learning environment.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Open to being mentored. Asks good questions. Learns from feedback.
  → To reach Proficient: Be proactive about asking for feedback. Take notes on what you learn.

  Proficient
    Helps newer team members. Shares knowledge willingly. Pairs effectively.
  → To reach Advanced: Offer to pair with someone newer. Share a useful technique each week.

  Advanced
    Actively mentors 1-2 people. Creates learning opportunities. Gives

  → To reach Expert: Take on a formal mentee. Set goals together and review regularly.

  Expert
    Defines the team's mentoring programme. Develops people systematically.
    Coaches across experience levels.


┌─────────────────────────────────────────────────────────────────────────────
│ Problem Solving & Initiative
│ Approaching problems systematically, showing initiative in finding solutions,
│ and not waiting to be told what to do.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Solves straightforward problems. Escalates appropriately when stuck.
  → To reach Proficient: When stuck, write down what you've tried before asking for help.

  Proficient
    Breaks down problems methodically. Tries multiple approaches before
    escalating. Shows initiative.
  → To reach Advanced: Before escalating, spend 30 minutes researching independently. Document your approach.

  Advanced
    Tackles ambiguous problems. Researches independently. Proposes solutions
    proactively.
  → To reach Expert: Look for problems that nobody owns. Propose a solution and drive it forward.

  Expert
    Solves cross-team systemic issues. Anticipates problems before they occur.
    Drives continuous improvement.


┌─────────────────────────────────────────────────────────────────────────────
│ Time Management & Prioritisation
│ Managing your workload effectively, prioritising competing demands, and
│ delivering reliably.
└─────────────────────────────────────────────────────────────────────────────

  Foundation
    Completes tasks on time with clear priorities set by others.
  → To reach Proficient: Use a task list daily. Track where your time actually goes for a week.

    Manages own time effectively. Balances project work with support.

  → To reach Advanced: Practice estimating tasks and comparing to actuals. Learn to say 'not right now' with a reason.

  → To reach Expert: Manage a shared backlog. Help the team prioritise collectively.

  Expert
    Optimises team workflow. Identifies and removes bottlenecks. Balances
    short-term delivery with long-term improvement.
